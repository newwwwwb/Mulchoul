# 물놀이 시설 작업 분배 · 현황 공유 웹앱 명세서

> 작성일: 2026-06-14 (rev.6)
> 상태: **전 항목 확정 · 스택 Supabase+Vercel — 개발 착수 가능**
> 본 문서는 개발 착수 전 합의된 요구사항을 정리한 것이다.

---

## 1. 프로젝트 개요

### 1.1 목적
물놀이 시설에서 근무하는 직원 7명이 매일 아침 출근 후, **자신의 번호·조·역할 배정을 한눈에 확인**하고, **현재 시각에 맞는 자신의 역할(문진/현장/휴식 또는 공통 활동)**을 실시간으로 파악할 수 있도록 돕는 웹앱이다.

### 1.2 해결하려는 문제
- 매일 랜덤으로 정해지는 번호·조·역할 배정을 구두/수기로 공유하면 헷갈린다.
- 특히 7번(휴식자 자리를 대행하는 인원)은 타임마다 자신이 어디서 무슨 일을 해야 하는지 혼동하기 쉽다.
- 누가 출근했는지, 언제 출근했는지 파악이 어렵다.

### 1.3 핵심 가치
- 여러 기기에서 **실시간 동기화**되는 단일 대시보드
- **이름 = ID** 원칙. 별도 비밀번호 없이 출석 화면에서 이름 클릭이 곧 본인 식별(로그인)
- 배정 과정(사다리/랜덤 추첨)은 사용자에게 노출하지 않고 **결과만** 표시

---

## 2. 사용자 및 규모

| 항목 | 내용 |
|---|---|
| 직원 수 | 7명 고정 (번호 1~7) |
| 권한 | 전원 동등 (관리자/일반 구분 없음) |
| 로그인 | 비밀번호 없음. 이름 = ID, 출석 화면에서 이름 클릭이 본인 식별 역할 |
| 대타 | 고정 직원이 결근 시 대타가 그 번호를 그대로 이어받음. 캘린더로 사전 입력, 기록 보관 (4.6 참고) |
| 동시 사용 | 여러 기기에서 동시 접속 및 실시간 갱신 필요 |

---

## 3. 사용자 시나리오

1. 직원이 출근 후 웹앱에 접속한다.
2. 화면에 표시된 자신의 이름을 클릭한다 → **출석 처리 + 출석 시각 기록.**
3. 대시보드에서 **오늘 자신의 번호와 조(A/B)**를 확인한다.
4. 근무 중 수시로 대시보드를 보며 **현재 시각 기준 자신의 역할**(문진/현장/휴식, 또는 준비·입장·정리·마감 등 공통 활동)을 확인한다.
5. 배정은 하루 1회 자동으로 랜덤 결정되어 있으며, 직원은 과정 없이 결과만 본다.

**대타 시나리오**
1. 고정 직원이 자신이 결근하는 날을 미리 캘린더에 입력한다 — 날짜 + 대타로 올 사람 이름.
2. 그날 대타로 온 사람이 출석 화면에서 **맡긴 직원의 이름을 대신 클릭**해 출석 처리한다.
3. 그 자리는 이름 뒤에 `(대타)`가 표시되고, 출석 시각도 기록된다.
4. 그 번호의 모든 역할 계산은 평소와 동일하게 진행된다(대타가 번호를 그대로 이어받음).
5. "언제 / 누가 맡겼고 / 누가 대타로 왔는지"가 기록으로 남아 조회 가능하다.

## 4. 기능 명세

### 4.1 출석 (Attendance)
- 앱의 **첫 화면**으로, 고정 직원 7명의 이름 목록을 표시한다.
- 이름을 클릭하면 해당 직원이 출석 처리되고, **출석 시각(HH:MM)**이 기록·표시된다.
- 이름 클릭은 출석인 동시에 **본인 식별(로그인)** 역할을 한다 (이름 = ID, 비밀번호 없음).
- 그날 대타가 잡힌 번호는, 대타로 온 사람이 **맡긴 직원의 이름을 대신 클릭**해 출석한다. 해당 자리는 이름 뒤 `(대타)`로 표시된다.
- 출석 상태는 모든 기기에 실시간 반영된다.
- **출석 취소(토글)**: 자기 이름을 다시 누르면 출석만 취소된다. 대타 예약은 유지된다.
- **미출석 표시**: 아직 출석을 누르지 않은 번호는 "미출석"으로 표시된다. 운영 시작 후에도 본인이 누르면 즉시 출석으로 전환되므로(지각 허용), "미출석"은 결근 낙인이 아니라 "아직 누르지 않음" 상태일 뿐이다.
- **결근**: 별도 결근 처리/메움 규칙은 두지 않는다(7.4 참고). 결근해도 역할 계산은 번호 기준 그대로이며, 화면에는 해당 번호가 "미출석"으로만 보인다.

### 4.2 일일 랜덤 배정 (Daily Assignment)
하루 1회, 다음을 **완전 랜덤**으로 결정한다. (사다리 타기는 예시 비유일 뿐, 구현은 단순 랜덤)

1. **번호 배정**: 직원 → 1~7번
2. **조 편성**: 1·2·3번 = A조 / 4·5·6번 = B조 / 7번 = 대행 인원
3. **첫 회차 역할**: 1회차에 A조가 문진인지 현장인지 결정 (B조는 반대)

- 배정은 **날짜 단위로 1회 고정**되며, 같은 날에는 변하지 않는다.
- 배정 과정 UI(추첨 버튼·애니메이션)는 **제공하지 않는다.** 결과만 대시보드에 표시.
- **생성 트리거**: Supabase **pg_cron으로 매일 자정(KST) 자동 생성**한다(상시 서버 불필요). 그날 `daily_records` 행이 없으면 번호 배정·firstRoundARole을 랜덤 생성하고, 주말이면 `busy_time`을 기본 3(2-1)으로 함께 세팅한다.
- **모든 항목 랜덤**: 번호 배정, 조 편성, firstRoundARole(1회차 A조 역할) 모두 매일 랜덤이다.
- **사전 노출**: 자정에 이미 확정되므로, 개장(09:00) 전에도 그날 번호·조·타임별 역할표를 대시보드에서 미리 볼 수 있다.

### 4.3 타임별 역할 자동 계산 (평일)
평일(수·목·금·월) 기준 로직. 회차·타임·역할·휴식이 아래 규칙으로 자동 계산된다.

**타임 구조**
- 1회차 운영 = 110분 = **50분 운영 + 10분 휴식 + 50분 운영** → 회차당 2개의 타임
- 평일 3회차 × 2타임 = **총 6타임**

**역할 스위칭 (회차 단위)**
- A조/B조의 문진·현장 역할은 **회차가 바뀔 때마다 스위칭**된다.
- 같은 회차 내 2개 타임은 역할이 동일하다.
- 예: 1회차 A=문진/B=현장 → 2회차 A=현장/B=문진 → 3회차 A=문진/B=현장

**휴식 로테이션 (타임 단위)**
- 매 타임마다 1→2→3→4→5→6번 순서로 한 명씩 휴식한다.
- 하루 6타임 동안 1~6번이 정확히 한 번씩 쉰다.
- **7번은 쉬지 않으며**, 그 타임에 쉬는 사람의 **역할과 조를 그대로 대행**한다.
- 결과적으로 평일 모든 타임은 **현장 3 + 문진 3**으로 유지된다.

### 4.3-2 타임별 역할 자동 계산 (주말·공휴일)
주말·공휴일(토·일)은 **4회차 = 8타임**으로 운영되며, 평일과 다른 휴식 규칙을 따른다.

**타임 구조**
- 평일과 동일하게 회차당 운영 110분 = 50분 + 휴식 10분 + 50분 → 2타임
- 4회차 × 2타임 = **총 8타임**

**역할 스위칭 (회차 단위)**
- 평일과 동일. 회차가 바뀔 때마다 A조·B조가 문진↔현장 스위칭.

**바쁜 타임 (하루 1개, 유동 지정)**
- 입장 직후라 문진 수요가 많은 "몇-1" 타임(각 회차 전반)이 주로 바쁘다.
- 하루에 **정확히 1개**의 타임을 "바쁜 타임"으로 지정한다(공정성 목적). 기본 후보는 2-1(T3)이며, 그날 다른 타임으로 변경 가능하다.
- 바쁜 타임에는 **아무도 쉬지 않고 7명 전원 투입**, 남는 1명은 **문진으로 보강**되어 그 타임만 **현장 3 + 문진 4**가 된다.

**휴식 로테이션 (타임 단위)**
- 바쁜 타임을 제외한 **7개 타임**에 1→2→3→4→5→6→7번이 순서대로 한 번씩 쉰다.
- 즉 주말에는 **7번도 한 번 쉰다**(평일과 차이점). 7번이 쉬는 타임은 1~6번이 그대로 3+3을 이루므로 대행이 필요 없다.
- 바쁜 타임이 어디냐에 따라 쉬는 순서가 자동으로 밀린다. (예: 바쁜 타임이 T5면 T1~T4에 1~4번, T5 전원, T6~T8에 5~7번)
- 바쁜 타임 외 모든 타임은 **현장 3 + 문진 3** 유지.

### 4.4 현재 역할 표시 (Now Role)
현재 시각이 시간표의 어느 구간에 속하는지 판별하여 표시한다.

- **운영 타임 중**: 각 직원의 역할 표시 — 문진 / 현장 / 휴식 (7번은 대행 중인 역할)
- **비운영 구간**: 전원 공통 활동명 표시 — 준비 / N회 입장 / 정리·점심 / 마감 등
- 시간표 데이터가 입력되어야 작동한다. (4.5-2 설정 참고)
- **오늘 타임별 역할표**: 현재 타임뿐 아니라, 오늘 하루 전체 타임(평일 T1~T6 / 주말 T1~T8)에서 본인이 맡을 역할을 한 번에 표로 보여준다. 현재 타임은 강조하고, 7번 대행·주말 보강·휴식도 함께 표시한다.

### 4.5 날짜 기반 평일/주말 자동 판별 (Day Type)
- 앱은 **오늘 날짜로 평일/주말(공휴일) 운영을 자동 판별**한다. 사용자가 모드를 수동 토글하지 않는다.
- **월~금 = 평일(3회차)**, **토·일 = 주말(4회차)**.
- **공휴일은 주말과 동일하게 4회차 운영**으로 처리한다.
- 공휴일 판별:
  - 프로토타입: 2026년 한국 공휴일 목록을 앱에 내장(대체공휴일 포함, 근로자의 날 제외).
  - **권장(본 개발)**: 공공데이터포털 "특일 정보 API"(한국천문연구원)로 공휴일을 자동 동기화. 음력 명절·임시·대체공휴일까지 정확히 반영. API 키 발급 및 백엔드/캐시 필요.
- 내장 목록 방식은 매년 갱신이 필요하며, 임시공휴일은 설정에서 수동 추가할 수 있도록 한다.

### 4.5-2 시간표 설정 (Timetable Config)
- 평일/주말 시간표는 **설정 화면에서 입력**하는 구조로 둔다.
- 본 문서 6장에 확정된 시간표를 기본값으로 사용한다.
- 향후 시간표 변경 시 코드 수정 없이 설정에서 갱신 가능하도록 한다.

### 4.6 대타 (Substitute)
- **원칙**: 대타는 별도 사용자 계정이 아니다. 시스템에 출석/식별되는 주체는 어디까지나 **결근하는 고정 직원의 이름**이며, 대타는 그 자리에 "기록"으로만 남는다.
- **입력 (캘린더)**: 고정 직원이 자신이 결근하는 날을 캘린더에 미리 등록한다. 입력 항목 = 날짜 + 대타로 올 사람 이름. (입력은 전원 가능, 권한 구분 없음)
- **당일 출석**: 대타로 온 사람이 출석 화면에서 맡긴 직원의 이름을 대신 클릭 → 출석 처리 + 출석 시각 기록 + 이름 뒤 `(대타)` 표시.
- **역할 계산**: 대타는 결근자의 번호를 그대로 이어받으므로, 번호·조·역할·휴식 로테이션 계산은 평소와 완전히 동일하다. (로직 변경 없음)
- **기록 보관**: 각 대타 건에 대해 날짜 / 맡긴 직원 / 대타로 온 사람 이름을 저장하여 추후 조회할 수 있도록 한다.
- **표시**: 대시보드의 전체 현황 및 해당 번호 영역에 `(대타)` 표식이 함께 노출된다.

## 5. 평일 배정표 (예시)

A조 = 1·2·3번, B조 = 4·5·6번, 7번 = 대행.
아래는 **1회차에 A조가 문진**으로 추첨된 경우의 예시다. (실제는 매일 랜덤)

| 회차 | 타임 | 시간 | 쉬는 사람 | A조(1·2·3) | B조(4·5·6) | 7번 대행 |
|---|---|---|---|---|---|---|
| 1회 | T1 | 10:00–10:50 | 1번 | 문진 | 현장 | 1번 자리(문진) |
| 1회 | T2 | 11:00–11:50 | 2번 | 문진 | 현장 | 2번 자리(문진) |
| 2회 | T3 | 13:15–14:05 | 3번 | 현장 | 문진 | 3번 자리(현장) |
| 2회 | T4 | 14:15–15:05 | 4번 | 현장 | 문진 | 4번 자리(문진) |
| 3회 | T5 | 15:50–16:40 | 5번 | 문진 | 현장 | 5번 자리(현장) |
| 3회 | T6 | 16:50–17:40 | 6번 | 문진 | 현장 | 6번 자리(현장) |

> 주의: 7번이 대행하는 역할은 "쉬는 사람이 속한 조의 그 회차 역할"이다. 예를 들어 T3에서 3번(A조)이 쉬고 A조가 현장이면, 7번은 현장을 수행한다.

## 6. 운영 시간표 (확정)

### 6.1 평일 (수·목·금·월) — 3회차
| 구분 | 시간 | 소요 | 내용 |
|---|---|---|---|
| 준비 | 09:00–09:30 | 30분 | 개장 준비(청소, 시설 점검) |
| 1회 입장 | 09:30–10:00 | 30분 | 1회 이용자 입장 |
| 1회 운영 | 10:00–11:50 | 110분 | 물놀이 운영 (T1 10:00–10:50 / 휴식 10:50–11:00 / T2 11:00–11:50) |
| 정리·점심 | 11:50–12:50 | 60분 | 1회 퇴장, 점심 |
| 2회 입장 | 12:50–13:15 | 25분 | 2회 입장, 부유물 청소 |
| 2회 운영 | 13:15–15:05 | 110분 | 물놀이 운영 (T3 13:15–14:05 / 휴식 14:05–14:15 / T4 14:15–15:05) |

## 7. 확정 내역

### 7.1 주말 8타임 휴식 로테이션 — **확정됨 (4.3-2 참고)**
- 하루 1개의 "바쁜 타임"을 유동 지정 → 그 타임은 7명 전원(현장 3 + 문진 4), 휴식 없음.
- 나머지 7개 타임에 1~7번이 순서대로 한 번씩 휴식(현장 3 + 문진 3).
- 주말엔 7번도 한 번 쉬며, 7번이 쉬는 타임은 대행이 불필요.
- 검증 완료: 바쁜 타임 위치와 무관하게 항상 현장 3 고정, 1~7번 공정 휴식.

### 7.2 대타 처리 — **확정됨 (4.6 참고)**
- 대타는 결근한 고정 직원의 번호를 그대로 이어받는다.
- 캘린더로 사전 입력, 당일 맡긴 직원 이름으로 출석(`(대타)` 표시), 대타 이력 기록.
- 역할 계산 로직은 변경 없음.

### 7.3 운영 세부사항 — **확정됨**
- **일일 배정 생성**: Supabase pg_cron으로 매일 자정(KST) 자동 생성(상시 서버 불필요). 번호·firstRoundARole 랜덤, 주말은 busy_time 기본 3 동시 세팅. 09:00 개장 전에도 미리 노출.
- **출석 취소·정정**: 자기 이름을 다시 누르면 출석만 취소(토글). 대타 예약(`substitutePlans`)은 그대로 유지된다. 대타 자리 출석은 대타 본인이 맡긴 직원 이름을 눌러 처리한다.
- **공휴일 인식**: 2026년 한국 공휴일 내장 목록으로 시작(4.5), 임시공휴일은 수동 추가. 추후 공공데이터 API로 전환.
- **주말 바쁜 타임 지정 (동기화)**:
  - 누구나 변경 가능(권한 구분 없음, 전원 동등 유지).
  - 변경 시 `daily_records.busy_time`에 저장 → Supabase Realtime으로 **전원 기기에 즉시 반영**.
  - 대시보드에 현재 바쁜 타임과 **마지막 변경자·변경 시각**을 표시하여 무단·무자각 변경으로 인한 혼선 방지.
  - "하루 1개" 규칙은 UI에서 단일 선택으로 강제.

### 7.4 결근·미출석 처리 — **확정됨**
- 결근 시 자리를 메우는 규칙은 두지 않는다. 역할 계산은 번호 기준 그대로 유지된다.
- 출석을 누르지 않은 번호는 "미출석"으로 표시하며, 운영 시작 후 본인이 누르면 즉시 출석으로 전환(지각 허용)된다.
- 즉 시스템은 "결근"을 별도 상태로 판정하지 않고 "미출석(아직 안 누름)"으로만 다룬다.

### 7.5 기술 표준 — **확정됨**
- **시간대**: 모든 시각·날짜 처리는 **Asia/Seoul(KST)** 기준. 자정 배정 생성, 날짜 판별, 출석 시각 기록, 스케줄러 모두 KST.
- **실시간 시계**: 대시보드는 1분 간격으로 현재 시각(KST)을 갱신하여 "지금 역할"을 자동 갱신한다. (프로토타입의 데모 슬라이더는 개발 시 실시간 시계로 대체)
- **운영 구간 구조 제약**: 모든 운영 회차는 **50분 + 휴식 10분 + 50분(=110분)** 구조를 전제한다. 시간표 설정 시 이 구조를 벗어나는 운영 시간은 지원 범위 밖이다.
- **데이터 보존**: `dailyRecords` 등 일자별 문서는 자동 삭제 없이 보관한다(7명 규모, 용량 무시 가능).
- **공휴일 유효기간**: 내장 공휴일은 2026년까지 유효. 2026-12-31 이후에는 "공휴일 데이터 갱신 필요" 안내를 표시한다.

## 9. 역할 계산 의사코드 (평일)

```
입력:
  now            = 현재 시각
  timetable      = 평일 시간표 (구간 목록)
  assignment     = { numberToEmployee, firstRoundARole }
  myNumber       = 내 번호 (1~7)

1. 현재 구간 찾기
   segment = timetable에서 start <= now < end 인 구간

2. 구간이 운영(operation)이 아니면:
   return segment.label   // "준비", "1회 입장", "정리·점심", "마감" 등

3. 운영 구간이면 → 타임 인덱스(T1~T6) 계산
   - 운영 110분을 [50분 운영 / 10분 휴식 / 50분 운영]으로 나눔
   - 앞 50분 → 해당 회차의 첫 타임, 뒤 50분 → 둘째 타임
   - 중간 10분(휴식 구간)이면 → return "휴식(전환)"  // 전체 공통
   - round 번호 + 앞/뒤로 타임 인덱스 t (1~6) 결정

4. 이번 타임에 쉬는 사람 = ((t - 1) mod 6) + 1   // 1→2→...→6
   restingNumber = t 번째로 쉬는 번호

5. 조 역할 결정
   roundIndex = 해당 타임의 회차 (1~3)
   if roundIndex 홀수: A조역할 = firstRoundARole
   else:               A조역할 = 반대(firstRoundARole)
   B조역할 = 반대(A조역할)

6. 내 역할 산출
   if myNumber == restingNumber:
       return "휴식"
   if myNumber in [1,2,3]:        // A조
       return A조역할
   if myNumber in [4,5,6]:        // B조
       return B조역할
   if myNumber == 7:              // 대행
       // 쉬는 사람의 조 역할을 대행
       if restingNumber in [1,2,3]: return A조역할
       else:                        return B조역할
```

### 9-2. 역할 계산 의사코드 (주말·공휴일)

```
입력:
  now, timetable(주말), assignment, myNumber
  busyTime = 그날 지정된 바쁜 타임 인덱스 (1~8, 기본 3)

1~3. 평일과 동일하게 현재 구간/타임 인덱스 t(1~8) 계산
     (비운영 구간이면 공통 활동명 반환, 휴식 전환 10분이면 "휴식(전환)")

4. 휴식자 매핑 생성 (하루 1회 계산)
   restNo = 1
   for t in 1..8:
       if t == busyTime: restOf[t] = 없음   // 전원 근무
       else:             restOf[t] = restNo; restNo += 1
   // 결과: 바쁜 타임 제외 7개 타임에 1~7번이 순서대로

5. 조 역할 결정 (평일과 동일)
   round = ceil(t / 2)
   A조역할 = (round 홀수) ? firstRoundARole : 반대
   B조역할 = 반대(A조역할)

6. 내 역할 산출
   resting = restOf[t]
   if t == busyTime:                      // 바쁜 타임: 전원 근무
       if myNumber in [1,2,3]: return A조역할
       if myNumber in [4,5,6]: return B조역할
       if myNumber == 7:       return "문진"   // 문진 보강 → 현장3 문진4
   else:                                  // 일반 타임: 1명 휴식
       if myNumber == resting: return "휴식"
       if myNumber in [1,2,3]: return A조역할
       if myNumber in [4,5,6]: return B조역할
       if myNumber == 7:
           // 7번이 쉬는 타임이면 위에서 이미 "휴식" 반환됨.
           // 그 외에는 쉬는 사람 자리 대행
           return (resting <= 3) ? A조역할 : B조역할
```

> 검증: busyTime을 1~8 어디로 두어도 모든 일반 타임은 현장 3 + 문진 3, 바쁜 타임은 현장 3 + 문진 4, 1~7번이 정확히 한 번씩 휴식함을 확인했다.

## 10. 화면 구성 (초안)

### 10.1 출석 화면 (첫 화면)
- 고정 직원 7명의 이름 목록 표시. 클릭 시 출석 + 출석 시각 기록 + 본인 식별.
- 그날 대타 예약이 있는 번호는 `(대타)` 표식과 함께 표시되며, 대타 온 사람이 맡긴 직원 이름을 클릭해 출석.
- 출석자/미출석자 구분 표시.

### 10.2 메인 대시보드 (단일 화면 중심)
- **상단**: 오늘 날짜, 날짜 기반 자동 판별 결과(요일 + 평일/주말·공휴일 + 회차), 현재 시각. 주말일 때만 "바쁜 타임" 선택 노출.
- **현재 역할 영역 (강조)**: 현재 시각 기준 큰 글씨로 "지금: 문진 / 현장 / 휴식 / 준비 …" 표시. 7번 대행·주말 보강 사유 부기.
- **내 배정 영역**: 내 번호, 내 조(A/B/7번), 출석 시각
- **오늘 타임별 역할표**: 오늘 전체 타임에서 내 역할을 한 줄로 표시(현재 타임 강조, 대행/보강/휴식 포함).
- **전체 현황 영역**: 7명 전원의 현재 역할을 한 번에 보여주는 표/그리드 (특히 7번 혼동 방지). 대타 자리는 `(대타)` 표식 노출.

### 10.3 대타 캘린더 화면
- 날짜를 선택해 "이날 ○○(고정 직원) 대신 △△(대타) 근무" 형태로 예약 입력.
- 기존 대타 예약 목록 조회·수정·삭제.
- 과거 대타 이력 조회 (날짜 / 맡긴 직원 / 대타 이름).

### 10.4 설정 화면
- 평일/주말 시간표 입력·수정
- (향후) 공휴일 처리

| 백엔드/DB | **Supabase (PostgreSQL)** | 관계형 모델 + 실시간 동기화 기본 제공. 별도 API 서버 불필요(프론트에서 직접 접근). 7명 규모는 무료 한도 내 |
| 실시간 동기화 | Supabase Realtime | 출석·배정·바쁜 타임 변경이 모든 기기에 즉시 반영(Postgres 변경 구독) |
| 일일 배정 스케줄러 | Supabase **pg_cron** (또는 Edge Function + cron) | 매일 자정(KST) 자동 배정 생성. 상시 서버 불필요 |
| 배포 | **Vercel** | React 배포 간단, 무료, GitHub 연동 자동 배포 |
| 상태 관리 | React 기본(useState/Context) | 소규모라 별도 라이브러리 불필요 |

**보안 비고**: 로그인이 없으므로(이름=ID) Supabase는 익명 접근을 허용하되, RLS(Row Level Security) 정책으로 테이블별 읽기/쓰기 범위를 제한한다. 민감 정보가 없고 7명 내부용이라 위험은 낮으나, 쓰기는 필요한 테이블에만 열어둔다.

**시간대 주의**: PostgreSQL/스케줄러는 기본 UTC이므로, 날짜 계산·pg_cron 실행 시각을 **Asia/Seoul(KST)** 기준으로 명시적으로 변환한다(자정 생성이 9시간 어긋나지 않도록).
